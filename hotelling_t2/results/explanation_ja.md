# Hotelling's T² による統計的評価の説明

## 1. この評価で行っていること

`mahalanobis_remake1` では、`train_data/train_sp500_us10y.csv` から作成した実データの参照windowを用いて、20個の stylized facts 特徴量の平均ベクトルと共分散行列を推定している。

Hotelling's T² 評価では、各 `each_mask/*.csv` から計算した20次元特徴量ベクトルが、この実データ参照分布から見てどの程度自然かを統計的に評価する。

ここでは、Mahalanobis距離と同じ特徴量、同じ通常の平均ベクトル、同じ通常の標本共分散行列を用いる。

## 2. 帰無仮説

この評価における帰無仮説は以下である。

```text
H0:
対象の each_mask データから計算した20次元特徴量ベクトルは、
train_data から作った実データ参照window群と同じ多変量分布に属している。
```

もう少し具体的には、以下のように考える。

```text
H0:
x_mask は、実データwindow特徴量の平均ベクトル μ と共分散行列 Σ によって表される
多変量正規分布から来た新規観測である。
```

つまり、対象データが「実データらしい特徴量分布の中に入っている」という仮説を置いている。

## 3. Hotelling's T² 統計量

各 `each_mask` データについて、20個の特徴量を並べたベクトルを `x` とする。

実データ参照window群から推定した平均ベクトルを `μ`、共分散行列を `Σ` とすると、Hotelling's T² 統計量は以下で計算する。

```text
T² = (x - μ)' Σ^{-1} (x - μ)
```

これは、共分散構造を考慮した多変量距離であり、Mahalanobis距離の二乗に対応する。

`T²` が大きいほど、対象データの特徴量ベクトルは実データ参照分布の中心から遠い。

## 4. p値の解釈

p値は、帰無仮説が正しいと仮定したときに、観測された `T²` と同程度またはそれ以上に外れた値が出る確率である。

```text
p値が大きい:
H0を棄却できない。
対象データは実データ分布から来たとしても不自然ではない。

p値が小さい:
H0を棄却する。
対象データは実データ分布から来たとは考えにくい。
生成データらしいと判断する。
```

注意点として、p値は「実データである確率」そのものではない。

例えば `p = 0.5` は、「実データである確率が50%」という意味ではない。正しくは、「もし対象データが実データ分布から来ているなら、今回以上に外れたT²統計量が出る確率が約50%ある」という意味である。

したがって、p値が大きいデータは「実データであると断定できる」のではなく、「実データ分布に属しているという仮説を棄却する理由がない」と解釈する。

## 5. 判定基準

本評価では、以下のように判定する。

```text
p >= 0.05:
実データ分布から来たという仮説を棄却しない。
実データらしい。

p < 0.05:
実データ分布から来たという仮説を棄却する。
生成データらしい。

p < 0.01:
より強く棄却する。
かなり生成データらしい。
```

## 6. 今回の結果の読み方

今回の結果では、以下の4つはp値が大きく、帰無仮説を棄却できなかった。

```text
mask2_brown.csv
mask4_brown.csv
mask2_sabr.csv
mask5_sabr.csv
```

これは、これらのデータが20個の特徴量空間において、実データ参照window群から見て自然な位置にあることを意味する。

一方で、それ以外のmaskはp値が非常に小さく、実データ参照分布から来たとは考えにくいと判断された。

## 7. 注意点

この評価は、以下の仮定に依存している。

- 20個の特徴量が実データらしさを十分に表現している
- 実データwindow特徴量が多変量正規分布に近い
- 通常の標本平均と標本共分散が安定して推定できている
- `train_data` から作成した参照windowが、判定対象の実データ分布を代表している

したがって、今回のHotelling's T²評価は強力な叩き台ではあるが、最終的には特徴量選択、robust covariance、shrinkage covariance、別手法との比較によって安定性を確認する必要がある。
